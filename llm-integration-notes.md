# 玥时之肤好评生成模型接入逻辑

## 0. 当前 DeepSeek 接入方式

本地页面现在通过 `server.mjs` 调用 DeepSeek，前端不会保存或暴露 API Key。

1. 复制 `.env.example` 为 `.env`
2. 在 `.env` 里填写 `DEEPSEEK_API_KEY`
3. 默认模型为 `deepseek-v4-flash`
4. 启动本地服务后访问 `http://127.0.0.1:5178/`

当前接口：
```text
POST /api/generate-review
```

前端只传用户选择的项目名；服务端会读取 `llm-project-knowledge.json`，把对应项目的定位、功效、包含服务和好评角度拼成提示词，再请求 DeepSeek。

DeepSeek 未配置、余额不足或网络异常时，页面会自动使用本地模板兜底，不影响门店现场使用。

## 1. 前端只传用户选择

前端页面只负责收集轻量信息：

- 门店 ID
- 体验项目 ID，最多 2 个
- 可选：顾客补充感受
- 可选：图片识别结果，不建议直接把图片作为默认主链路

示例：

```json
{
  "storeId": "yueshi-demo",
  "projectIds": ["clean-7", "balance-9"],
  "customerNote": "鼻翼黑头多，做完没有紧绷"
}
```

## 2. 后端按项目 ID 读取知识表

后端从 `llm-project-knowledge.json` 里取出用户选择的项目，拼成精简上下文。

不要每次把完整培训课件原文传给模型。课件是知识来源，线上推理只传选中项目相关字段：

- `displayName`
- `positioning`
- `suitableFor`
- `coreSteps`
- `ingredients`
- `effects`
- `includedService`
- `reviewAngles`

## 3. 推荐系统提示词

```text
你是玥时之肤门店的好评文案助手。
请根据顾客体验项目生成一条自然真实的美团丽人好评。

要求：
1. 文字 120-180 字。
2. 语气像真实顾客，不要广告腔，不要像模板。
3. 必须自然覆盖：项目效果、服务细节、门店环境。
4. 最多围绕 2 个项目写，不要堆砌全部成分。
5. 成分和功效只作为背景知识，转化成顾客能感知的体验。
6. 必须有 1-2 个常见 emoji，比如 😊、✨、👍、🌿，但不要堆表情。
7. 禁止写好评返现、送项目、折扣、微信、小红书、私下交易。
8. 禁止根治、永久、百分百、立刻变白、医美治疗等绝对化表达。
9. 禁止姐妹、种草、强推、闭眼冲、宝子等营销口吻。
10. 不要出现大量符号、编号、标题。
11. 换一篇时要参考上一版，避开相同开头、相同句式和相同 emoji。
12. 真实细节只围绕已提供的项目、按摩、服务、干净安静的环境来写，不要编造香薰、茶水、音乐、具体装修等未提供信息。
```

## 4. 用户提示词模板

```text
门店：{{storeName}}
体验项目：
{{selectedProjectKnowledge}}

顾客补充：
{{customerNote}}

请输出一条可直接复制到美团/大众点评的自然好评。
只输出正文，不要解释。
```

## 5. 为什么这样设计

前端展示的是顾客能理解的项目名，后端保存的是模型能理解的项目知识。

这样有三个好处：

- 顾客不会被成分名和操作步骤吓到。
- 模型知道每个项目的产品、成分、功效和真实体验角度。
- 每次只传 1-2 个项目的知识，token 成本比整份课件低很多。

## 6. 图片的处理方式

图片不建议作为默认必填。

如果后面要用图片，建议流程是：

1. 多模态模型先把图片转成文字观察，例如“房间干净、仪器在旁边、做完皮肤泛红不明显”。
2. 把这个观察结果作为 `customerNote` 的一部分交给 DeepSeek。
3. 最终好评仍由 DeepSeek 生成。

这样可以保留 DeepSeek 的低成本和文案自然度，同时用多模态模型做辅助。
